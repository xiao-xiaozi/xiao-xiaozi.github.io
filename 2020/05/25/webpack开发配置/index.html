webpack开发配置

2020-05-25

前言

记录一下webpack的开发配置。

详解配置

基本结构

webpack.config.js的最简构成（在开发环境下）大概聚合了下面这几项配置。其中

  • entry 是入口文件配置，一个指向入口文件的路径
  • output 是输出配置，指定了输出文件名和位置
  • module 主要使用了module的rules属性，用于配置各式loader
  • plugins 用于配置插件
  • mode 用于指定当前模式，有development和production两个值
  • devServer 只用于开发模式，用于启动一个服务便于实时查看效果

入口配置

entry指定一个入口文件的路径，webpack根据该入口文件进行分析，打包依赖。

输出配置

output指定输出文件名和输出到哪里。filename属性前可以加一个js指定输出文件夹。若不指定则会输出到根文件夹，在同时包含字体文件，样式文件时根文件夹会过于混乱，建议指定输出文件夹。

rules配置

module的rules属性用于配置各种loader，将对应的资源文件转为webpack能够识别的js文件。

样式资源

样式资源包含了CSS、Less、Sass等CSS预处理语言。使用loader时需要线通过npm安装。

test使用正则匹配相对应的文件后缀名，然后对匹配到的文件使用use中的loader，loader的执行顺序是从右往左。
loader说明：

  • css-loader 用于将css文件转为commonJS模块加载到js中。
  • style-loader 用于创建style标签，将js中的样式资源插入到html文件的header中。

预处理语言
CSS预处理语言的loader配置，仅需要在CSS loader配置中的use数组加上对用的预处理语言的loader即可，less配置见下。

图片资源

图片资源分CSS中的图片资源和html中的图片资源，对对应的资源使用不同的loader。
CSS中的图片资源

在单个loader的情况下可以直接使用loader:'url-loader'配置。options属性对loader所使用的loader进行设置。此处：

  • limit 图片大小小于limit设置的值时，图片会被base64处理
  • name 对输出图片名字进行配置，默认以hash值为名输出，名字较长，此处配置以hash值前10位为名，带上文件后缀名输出即可。

html中的图片资源

html文件中的图片资源使用html-loader解析。若出现图片资源引入路劲出错问题，有可能时url-loader使用了es6模块化的原因。可以在url-loader的配置对象中新增属性esModule:false,解决该问题。作者在使用过程未遇到此问题，有遇到的可以交流下。

其它资源

在开发过程中，可能还包含除了JS、CSS、图片资源以外的资源，比如字体文件，icon等。此类资源使用file-loader处理。

exclude属性表示排除，排除掉正则匹配到的文件以外的文件。

plugins配置

  1. 自动生成html的插件。

可以通过传一个template属性指定生成html文件的模板，若不配置模板，则自动生成。

devtool配置

配置source-map的选项。source-map 提供源代码到构建后代码映射技术，如果构建后代码出错了，通过映射可以追踪到源代码错误。
该项配置有多个可选项，每个可选项映射的结果不一样，一般使用inline-source-map。其它可选项详见wenpack官网说明。

mode配置

模式配置，只有两个模式可选生产模式（production）和开发模式（development）。

devServer配置

devServer开发服务器配置，用来自动化（自动编译、自动打开浏览器、自动给刷新浏览器）。只会在内存中编译打包，不会有任何输出。
